📌 项目名称
问答互动板（Q&A Board）

🎯 核心目标
构建一个纯前端、无依赖、可离线运行的交互式问答管理工具，支持用户初始化数据、编辑问题、拖拽排序，并持久化保存状态。

✅ 功能需求清单
1. 数据初始化
   按钮名称：初始化数据
   点击后展开两个选项：
   本地导入：从用户设备选择 .json 文件（兼容现代 File System Access API 和传统 <input type="file">）
   URL 导入：输入公开 JSON 链接，自动加载远程数据
   数据格式要求：
   json
   {
   "data": [
   { "msg": "问题1", "weight": 1, "date": "2025-01-01 12:00:00" },
   ...
   ]
   }
   自动过滤空/无效条目，保留 msg、weight、date 字段
2. 问题管理
   添加问题：输入框 + 回车/按钮提交，自动分配 weight = max + 1
   编辑问题：点击“编辑” → 显示输入框 → “保存”更新内容
   删除问题：逐条删除，带确认提示（清空时全局确认）
   实时校验：编辑时仅当内容变化才显示“保存/取消”按钮
3. 拖拽排序
   使用 SortableJS 实现原生拖拽（非 React）
   拖动手柄：⋮⋮ 图标
   拖拽完成后：
   自动重排数组顺序
   更新所有项的 weight 为 1, 2, 3, ...（按新顺序）
   触发本地存储保存
4. 数据持久化
   自动保存：任何增删改拖操作后，同步写入 localStorage
   状态恢复：页面刷新后自动加载上次数据
   源文件追踪：若通过本地文件导入，记录 FileHandle 用于后续同步
5. 数据导出与同步
   导出新文件：生成标准 JSON 文件下载（qa_questions_export.json）
   同步源文件：
   若存在有效 FileHandle，直接覆写原文件
   否则尝试另存为（现代浏览器）或降级为下载（提示用户手动替换）
6. UI/UX 要求
   响应式布局，适配桌面与平板
   卡片式设计 + 圆角 + 阴影，视觉清爽
   操作反馈：
   底部 Toast 提示（成功/错误信息，3 秒自动消失）
   按钮悬停动效（轻微上浮 + 阴影增强）
   空状态提示：“暂无问题，请初始化数据或添加”
7. 技术约束
   纯 HTML/CSS/JS，不使用任何框架（如 React/Vue）
   不依赖 dnd-kit（因其为 React 专用），改用 SortableJS
   兼容主流现代浏览器（Chrome/Firefox/Edge 最新版）
   所有资源通过 CDN 引入（仅 SortableJS）

🚫 非功能需求（排除项）
❌ 不支持用户登录/多端同步
❌ 不处理图片/富文本（仅纯文本问题）
❌ 不验证 URL 安全性（用户自行确保链接可信）
